## Mod specification

Create a mod that turns the game into an extraction shooter. Rules:

- **Save restriction**: the player can only save within the limits of a friendly base
  area.
- **Single save file**: one save file is used and overwritten for all saves —
  quicksaves, autosaves, and regular manual saves from the menu.
- **Death**: on death the player's entire inventory is dropped as a backpack stash at
  the location of death. The player then respawns at the last friendly base area they
  entered and an autosave is made.
- **Night respawn**: never respawn the player at night (20:00 – 08:00). If the player
  dies within those hours, respawn at 08:00.

## Documentation gaps found in this test run

These are things the test author needed to know that are **not covered** by the current
guide and required reading community mod source code to discover:

| # | Gap | Resolved by | Impact if unresolved |
|---|-----|-------------|----------------------|
| GAP-1 | `exec_console_cmd` not documented | Jabbers Soulslike mod source | Cannot trigger saves to a specific filename or autosave after respawn |
| GAP-2 | Stash/backpack container section name not documented | Jabbers Soulslike mod source (`"inv_backpack"` confirmed) | Cannot spawn the correct container at the death location |
| GAP-3 | `ui_load_dialog.delete_save_game(name)` not documented | Jabbers Soulslike source | Required to delete a just-written save file; combined with `on_console_execute` this catches menu saves and autosaves |
| GAP-4 | `db.actor_inside_zones` table not documented | Fair Fast Travel mod source | Must use less-precise distance check instead of zone-entry events |
| GAP-5 | Level name strings (`l01_escape`, `k00_marsh`, etc.) not listed in guide | Partially from guide examples + inference from smart terrain prefixes | Zone positions can't be mapped to correct levels without this |
| GAP-6 | `actor_on_before_death` requires modded exes — vanilla death interception not documented | Callbacks reference marks it **Exes** | No vanilla fallback implemented |
